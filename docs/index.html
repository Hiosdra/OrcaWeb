What it does

A full 3D-printing slicer that runs entirely client-side, powered by WebAssembly.

🔒

100% Private

Your STL files never leave your browser. No accounts, no upload, no cloud.

⚡

OrcaSlicer Engine

v2.3.1 core compiled to WebAssembly via Emscripten. Same slicing quality as the desktop app.

🎨

G-code Visualiser

Layer-by-layer toolpath preview with a slider. Model and G-code rendered in the same coordinate system.

📦

Profile Import

Drag in any OrcaSlicer machine, filament, or process JSON profile from your desktop installation.

🖨️

Printer Presets

Built-in presets for Bambu Lab P1S / X1C, Prusa MK4, Creality Ender 3, Voron 2.4, and generic 0.4/0.6 mm.

💻

CLI Tool

Node.js command-line interface for batch slicing, headless pipelines, and CI workflows.

Architecture

Slicing runs in a Web Worker to keep the UI responsive. The WASM module is loaded once per session.

Browser
├── main thread (React)
│   ├── ModelViewer  — Three.js, STL on print bed
│   ├── SettingsPanel — presets + profile import
│   ├── GcodeViewer  — toolpaths layer by layer
│   └── worker-singleton.ts  — single Worker instance
│
└── Web Worker
    ├── slicer.worker.ts
    │   └── wasm-loader.ts  (_orc_init / _orc_slice)
    └── public/wasm/
        ├── slicer.js    (1.2 MB, Emscripten glue)
        ├── slicer.wasm  (6.4 MB, OrcaSlicer core)
        └── slicer.data  (144 MB, profiles + data)

Quick setup

The WASM artifacts are not stored in the repo (144 MB). Download them once with the setup script.

  1. Clone the repo git clone https://github.com/Hiosdra/OrcaWeb.git && cd OrcaWeb
  2. Install dependencies npm install
  3. Download WASM artifacts ~150 MB, one-time node scripts/download-wasm.mjs
  4. Start dev server npm run dev
