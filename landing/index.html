Features

Everything the desktop app does,
nothing leaves your machine

OrcaWeb ships the same slicing engine that powers the desktop app — compiled to WebAssembly, no compromises.

100% private

Every model, profile, and G-code file is processed entirely in your browser. Nothing is ever uploaded to a server.

Real OrcaSlicer engine

OrcaSlicer v2.4.0 with full feature parity — fuzzy skin, every infill pattern, supports, every parameter. Same G-code quality as desktop.

STL, 3MF, OBJ & STEP

CAD-native STEP import runs on OCCT, compiled straight into the engine — no separate conversion step, no round trip.

G-code visualiser

Layer-by-layer toolpaths rendered in Three.js, alongside the 3D model preview, in one shared coordinate system.

Your profiles, your presets

Import machine, filament, and process JSON profiles straight from a desktop OrcaSlicer install, or start from built-in presets for Bambu Lab P1S/X1C, Prusa MK4, Ender 3, and Voron 2.4.

Installable & offline

A proper PWA — after the first ~29 MB load, everything is cached, so OrcaWeb keeps slicing with no connection at all.

How it works

Model to G-code, four steps

No install, no account creation — open the app and go.

01

Drop a model

STL, 3MF, OBJ, or STEP — drag it in for an instant 3D preview on a real-scale print bed.

02

Configure

Pick a built-in printer preset, or import your own machine, filament, and process profiles.

03

Slice

The engine runs in a Web Worker and returns G-code in seconds. The UI never freezes.

04

Inspect & download

Scrub through layers in the G-code visualiser, then download a file that's ready for your printer.

Under the hood

Built from the real thing

OrcaWeb compiles the actual C++ OrcaSlicer core to WebAssembly with Emscripten — it isn't a reimplementation. The engine runs off the main thread in a Web Worker, and there's no 200 MB resource preload: the whole engine, OCCT included, is roughly 29 MB.
